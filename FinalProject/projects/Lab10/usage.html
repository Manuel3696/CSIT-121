How is it used?

The use of XML Signature Properties requires existing XML documents that previously underwent XML digital signature processing. The <SignatureProperties> element allows developers to embed these properties before signatures reference them. The addition of SigningTime as a property stands among the primary applications of XML signature elements as a means of determining the exact time a document was signed.

A SigningTime property usage example is as shown below:

Example 1

                <SignatureProperties>
                  <SignatureProperty Target="#Signature1">
                    <SigningTime>2025-05-06T10:00:00Z</SigningTime>
                  </SignatureProperty>
                </SignatureProperties>

In the demonstration above, the signature property is linked to Signature1 specifically and the system records signature timestamps simultaneously. Other examples on how to specify signature creation policy details entail using <SignaturePolicyIdentifier> and match signer roles by inputting <SignerRole> data. These elements describe policy within which the signature was established and designation or signer position (e.g "legal officer" or "approver"), respectively.

Another usage is in encrypting XML element content. For instance, it may be useful for intermediate agents to know that John used a credit card with a particular limit, but not the card's number, issuer, and expiration date. In this case, the content (character data or children elements) of the CreditCard element can be encrypted, as sown in Example 2 below:

Example 2

                <?xml version="1.0"?> 

                <PaymentInfo xmlns="http://example.org/paymentv2">
                    <Name>John Smith</Name>
                    <CreditCard Limit="5,000" Currency="USD">
                        <EncryptedData xmlns="http://www.w3.org/2001/04/xmlenc#"
                                    Type="http://www.w3.org/2001/04/xmlenc#Content">
                            <CipherData>
                                <CipherValue>A23B45C56</CipherValue>
                            </CipherData>
                        </EncryptedData>
                    </CreditCard>
                </PaymentInfo>

Other usages are within digital contracts where these properties help to establish verified date and signature data that fulfills legal requirements. Also, these signature elements support machine-to-machine interactions including automated business workflows that require systems to both verify the information and check the signature's relevance.
